Provides virtual scrolling for efficient rendering of large documents.

## Usage

```ts
const {
  visiblePages,
  isVirtualScrollEnabled,
  setContainerRef,
  scrollToPage,
} = usePdfKitVirtualScroll()
```

## Options

```ts
usePdfKitVirtualScroll({
  totalPages: 100,
  threshold: 10,
  overscan: 2,
})
```

| Option | Type | Default | Description |
|--------|------|---------|-------------|
| `totalPages` | `number` | `0` | Total number of pages |
| `threshold` | `number` | `10` | Minimum pages to enable virtual scroll |
| `overscan` | `number` | `2` | Pages to render outside viewport |

## Return Values

| Property | Type | Description |
|----------|------|-------------|
| `visiblePages` | `Ref<number[]>` | Pages to render |
| `isVirtualScrollEnabled` | `ComputedRef<boolean>` | Virtual scroll active |
| `setContainerRef` | `(el: HTMLElement) => void` | Set scroll container |
| `scrollToPage` | `(page: number) => void` | Scroll to page |

## Example

```vue
<template>
  <div ref="containerRef" class="h-[600px] overflow-auto">
    <div v-for="page in visiblePages" :key="page">
      <!-- Render page {{ page }} -->
    </div>
  </div>
</template>

<script setup>
const containerRef = ref(null)

const {
  visiblePages,
  isVirtualScrollEnabled,
  setContainerRef,
} = usePdfKitVirtualScroll({
  totalPages: 100,
  threshold: 10,
})

onMounted(() => {
  if (containerRef.value) {
    setContainerRef(containerRef.value)
  }
})
</script>
```

## How It Works

Virtual scrolling only renders pages that are visible in the viewport plus a few extra pages (overscan) for smooth scrolling. This significantly improves performance for documents with many pages.

::tip
Virtual scrolling is enabled by default in `NuxtPdfKit` when the document has more pages than the threshold.
::
